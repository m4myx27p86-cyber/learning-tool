# 編集前・編集後の観点別比較

| 観点 | 編集前 | 編集後 |
|---|---|---|
| 1. エフェクト | 正解時のフィードバックカードと簡易コンフェッティあり | 既存エフェクトを維持。誤り発見に正解した段階でも確認カードを表示し、修正入力へ自然に誘導 |
| 2. ファイルの整理 | Speaking Reviewのセクションが日付＋内容で細かく分散 | CSV上のsectionを問題形式別に整理：Photo description / Responding question / Opinion / E-mail writing / Read aloud |
| 3. 間違えを解きなおす機能 | 語順並べ替え・穴埋めで誤答チャンクを復習 | 新教材「誤り発見→修正」を追加。Which phrase is incorrect? → 正解したら修正入力へ進む2段階式 |
| 4. 操作性 | 語順並べ替えでも入力欄フォーカスによりスマホキーボードが出る可能性 | TOEIC Speaking復習（語順並べ替え）はreadonly＋inputmode=noneにし、語句タップだけで回答可能 |
| 5. 動作の重さ | 既存の単一script.jsに機能が集中 | 大きな構造変更は避け、読み込み関数と表示関数を最小追加。既存教材の読み込み経路は維持 |
| 6. 見た目・スマホ対応 | 既存のレスポンシブUIあり | 誤り発見用カードと選択肢グリッドを追加。スマホでは1列表示に自動変更 |
| 7. 抜本的な解決を必要とする問題点 | 問題数が増えるほどscript.jsが肥大化 | 将来的には教材ごとにJS/CSVを分割し、教材データをmanifest管理すると保守しやすい |
| 8. 細かい問題点 | セクション名が日付依存で、復習時に探しにくい | 形式別セクションに統一したため、問題選択画面で目的別に選びやすい |
| 9. 新たな機能の提案 | 語順・穴埋め中心 | 誤りパターン別フィルター、AI風フィードバック文のテンプレート化、ミス頻度ランキングの追加が有効 |
