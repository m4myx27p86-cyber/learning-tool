## 4. 現在のサイトで注意すべき問題
- [ ] GitHub Pages にCSVファイルが配置されていない教材は、教材カードが表示されても読み込みエラーになる可能性がある。
- [ ] Apps Scriptを更新しても「新しいデプロイ」または「デプロイを管理→新バージョン反映」をしないと、ブラウザ側の確認バナーが警告になる可能性がある。
- [ ] `GAS_WEB_APP_URL` が古いデプロイURLの場合、送信先が古い処理のままになる可能性がある。
- [ ] no-cors送信はブラウザ側で詳細な成功内容を読めないため、最終確認はスプレッドシート側で行う必要がある。
- [ ] localStorageは端末・ブラウザごとの保存なので、別端末ではホームの植物・履歴・カレンダーが共有されない。
- [ ] 教材数が増えたため、CSVファイルの場所・ファイル名・`TEST_CONFIG` の対応を今後もチェックする必要がある。
- [ ] 音声はブラウザの自動再生制限により、ユーザー操作後にのみ鳴る場合がある。
- [ ] iPhone/Safariではキャッシュが残りやすいため、CSS/JS更新後に強制再読み込みが必要な場合がある。
